Everyday Git

What is Git?

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Version control systems are a category of software tools that helps in recording changes made to files by keeping a track of modifications done in the code. By maintaining a history of changes, a VCS enables developers to manage their projects more efficiently, whether they are working alone or as part of a team. There are several version control systems available, such as Subversion (SVN), Mercurial, and Perforce, this blog will focus on Git. Git is widely used due to its distributed nature and robust feature set, making it a popular choice for developers around the globe. We will also explore GitHub, a platform that integrates with Git to provide a collaborative environment for managing and sharing Git repositories. In this blog I will be diving deeper into how Git and GitHub work and how they can enhance your development workflow.

Installing Git

To check if Git is already installed on your machine,open your terminal or command prompt and type git --version. If Git is not installed, download the installer from the official Git website and follow the installation instructions for your operating system. Once installed, verify the installation by running git --version

again to ensure Git is correctly set up on your system.

Configuring Git

Set up Git with your personal username and email address.

Follow the steps below:
  1. Open your terminal or command prompt.
  2. Set your username by typing:
    git config --global user.name "Your Name"



  3. Set your email address by typing:
    git config --global user.email "your.email@example.com"



  4. Verify your configuration by running:
    git config --global --list

Intializing a Git repository

To start using Git for your project, you need to initialize a Git repository.
This command creates a hidden .git directory in your project folder, which contains all the necessary files for version control. Here’s how you do it:

  1. Navigate to your project folder in the terminal or command prompt

  2. Initialize the repository by
    git init

  3. Verify the repository setup by listing the contents :
    ls -a

Adding files to the staging area

In Git, the staging area is an intermediate space where you prepare your files before committing them to the repository. It acts as a buffer between your working directory and the repository, allowing you to review and organize changes before finalizing them.

Steps to Add Files to the Staging Area
  1. Check the status of your working directory to see which files are untracked or modified:
    git status

    This command will list files that have been changed but not yet added to the staging area
  2. Add files to the staging area by using:
    git add filename


    or
    Replace filename with the name of the file you want to stage. To add all files in the directory, you can use: git add .

  3. Verify staged files by running:
    b git status

Commiting changes

Committing changes is a fundamental step in using Git. A commit is like a snapshot of your project at a particular point in time. It captures the state of your files in the staging area and records it in the repository's history.


How to Commit Changes

  1. Stage Your Changes: Before you can commit, you need to add your changes to the staging area.
    git add filename

    Or add all changes: git add .


  2. Commit Your Changes: To commit, use the following command:
    git commit -m "Your commit message here"

Branches

In Git, a branch is like a separate workspace where you can make changes without affecting the main project. Branches are important because they allow you to:

  • Work Independently: Develop new features or fix bugs without disrupting the main code.
  • Collaborate Efficiently: Multiple team members can work on different tasks simultaneously.
  • Experiment Safely: Try out new ideas without risking the stability of your main project.

Branches help keep your code organized and manageable, making development smoother and more collaborative.

Creating a branch

Creating a branch in Git allows you to work on a new feature or fix independently from the main codebase.

Here's how to create a new branch:
  1. Open your terminal or command prompt.
  2. Create a new branch by typing:
    git branch branch_name Replace branch_name with the name you want for your branch.
  3. Switch to the new branch by using: br git checkout branch_name


    Alternatively, you can create and switch to the branch in one step with:
    git checkout -b branch_name

  4. Verify your branch by running:
    git branch

Viewing all branches

To manage your branches effectively, you need to know how to view all branches in your repository and identify the currently active branch. Here’s how to do it:

  1. View All Branches: To see a list of all branches in your repository, use:
    git branch

  2. View Remote Branches: If you want to see branches that exist on a remote repository, use:
    git branch -r

  3. View Both Local and Remote Branches: To view all branches, both local and remote, use:
    git branch -a

Switching active branches

Switching branches in Git allows you to move between different lines of development in your project. Here’s how you can change your active branch:

  1. Open your terminal or command prompt.
  2. Switch to another branch by typing:
    git checkout branch_name
  3. Alternatively, use Git Switch (if available) for a simpler command:
    git switch branch_name
  4. Verify the active branch by running:
    git branch

Merging a branch

Merging branches in Git allows you to combine changes from one branch into another, typically bringing updates from a feature branch into the main branch. Here’s how to do it:

  1. Switch to the branch you want to merge changes into. For example, if you want to merge changes into the main branch, first switch to it:
    git checkout main
  2. Merge the desired branch into your current branch. For instance, to merge changes from feature_branch into main, use:
    git merge feature_branch
  3. Resolve any conflicts if they arise. Git will notify you of conflicts that need to be resolved manually before completing the merge.
  4. Commit the merge if there were conflicts resolved manually:
    git commit -m "Merged feature_branch into main"

Deleting a branch

Deleting branches helps keep your repository clean by removing branches that are no longer needed. Here’s how to delete both local and remote branches:

  1. Open your terminal or command prompt.
  2. Delete a local branch by typing:
    git branch -d branch_name

    The -d flag ensures the branch is only deleted if it has been fully merged. Use -D to force delete a branch, even if it hasn’t been merged:
    git branch -D branch_name
  3. Verify the deletion by running:
    git branch
    This command will list all remaining branches.

Pushing Changes to GitHub Using Git

To push your local changes to a GitHub repository, follow these steps:

  1. Ensure Your Local Repository Is Connected to GitHub First, make sure your local repository is linked to a GitHub repository. You can check this with:
    git remote -v

    This command lists the remote repositories associated with your local repository. If you need to add a remote, use:
    git remote add origin https://github.com/username/repository.git
  2. Stage and Commit Your Changes Before pushing, ensure you have committed your changes:
    git add .

    This stages all changes. Then commit them with:
    git commit -m "Your commit message"
  3. Push Changes to GitHub Push your committed changes to GitHub :
    git push origin branch_name

    Replace branch_name with the name of the branch you want to push to (e.g., main or master).

  4. Check your GitHub repository online to confirm that your changes have been uploaded.

Next Steps in Your Git Journey

Now that you’ve learned how to use Git for basic version control and pushing changes to GitHub, here are some recommended next steps to deepen your understanding and enhance your workflow:

  1. Learn About Branch Merging and Rebasing: Understanding how to merge branches and use rebasing techniques will help you manage complex workflows and maintain a clean project history.

  2. Explore Advanced Git Commands: Familiarize yourself with advanced Git commands like git stash , git cherry-pick, and git revert. These commands offer additional flexibility for managing your changes and handling various development scenarios.
  3. Master GitHub Features: Dive into GitHub features such as Pull Requests, Issues, and GitHub Actions.
  4. Integrate with Git Tools: Explore tools that integrate with Git, such as Git GUI clients (e.g., SourceTree, GitKraken) and continuous integration services (e.g., Travis CI, CircleCI).
  5. Collaborate with Git: Practice using Git in collaborative environments by contributing to open-source projects or working on team projects.

By continuing to explore these areas, you'll build a stronger foundation in version control and collaboration, making you more effective and efficient in managing your codebase.
